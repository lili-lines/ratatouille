The AS5600 angle sensor can output its value in 3 ways :

| Mode | What the chip outputs | How you read it |
|---|---|---|
| **PWM** | a pulse whose width = the angle | a timer pin |
| **I2C** | a digital number in a register | the I2C bus |
| **Analog**🐭 | a voltage (0–3.3 V) = the angle | an ADC (Analog-to-Digital Converter) pin |

**PWM (Pulse Width Modulation)** = a signal switched on/off very fast. The angle is encoded by the duration of the "on" time, not by the voltage level. He gives the full angle, but to read it you must time a whole pulse, so each reading waits one full cycle → slow read speed and a low sampling rate (few readings /s).

**I2C (Inter-Integrated Circuit)** = The 2x AS5600 has a fixed same I2C address (0x36) that you cannot change. So two sensors (left + right whisker) cannot share one I2C bus, clash or need an extra switching chip = a multiplexer.

🐭 **Analog** = each sensor gets its own wire to its own ADC pin. Two sensors = two pins, no clash. Also analog is faster the read is instant, so the robot reacts quickly when a whisker hits a wall. And easy to clean: it is an analog signal, so a simple RC filter removes the motor noise. <br>
RC = a **R**esistor + a **C**apacitor wired together, it lets the slow useful signal pass and blocks the fast noise.
